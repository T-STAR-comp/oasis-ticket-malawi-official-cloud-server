SHIP TO PRODUCTION (before git push)
------------------------------------
npm run ship
(uses 8GB Node heap for frontend build — close other apps if OOM)

This builds:
  - Main app     → public/client/
  - Admin panel  → public/admin/   (https://ticketmalawi.com/admin/)
  - API          → dist/

Commit and push public/client/, public/admin/, and dist/ with your code changes.

DATABASE (after deploy on production)
-------------------------------------
npm run db:migrate:service-fee-settings
(and any other pending migrations)

VERIFY DEPLOY
-------------
GET https://ticketmalawi.com/api/health
  apiVersion should be 3+
  features.financeSettings should be true
  features.adminUi should be true

Admin panel URL: https://ticketmalawi.com/admin/

CPANEL / SPACESHIP
------------------
git pull in oasis-ticket-malawi-official-cloud-server
npm install --omit=dev
(create .env from .env.example)
Restart Node app (startup file: server.cjs)

No build step needed on the server if you ran ship before pushing.
